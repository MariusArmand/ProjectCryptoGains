Getting Started

1. Configure Settings

Go to Configure => Settings

Fiat currency

Select the fiat currency you wish to use for conversions in Project Crypto Gains (PCG)

CoinDesk Data API key

PCG uses the CoinDesk Data API for conversions to fiat; Go to the CoinDesk Data website to sign up for an API key that suits your needs

2. Import source data

Go to
  • Sources => Manual Ledgers*
  • Sources => Kraken Ledgers

*optional

Each import window has its own help section; Assets unknown to PCG will need to be configured via
  • Configure => Asset Catalog
  • Configure => Kraken Assets

3. Process

3.1 Refresh Ledgers

Go to Process => Ledgers to refresh PCG ledgers

3.2 Refresh Trades

Go to Process => Trades to refresh PCG trades

- PCG uses ledgers and trades as input for generating other reports (e.g., Gains, Rewards, etc.) - After importing new source data, always refresh the PCG ledgers and trades (in that order), before refreshing any other reports..

FAQ

Why are my airdrops in from my Kraken ledgers csv not picked processed

In the Kraken ledgers CSV, airdrops are identified by the type 'transfer' and subtype 'spotfromfutures', but these attributes may not unique to airdrops; Hence, PCG does not process ledger entries with the type 'transfer' For airdrops, a manual ledger entry should be made and uploaded via Sources => Manual Ledgers

Warnings Explained

Retrieved 0.00 exchange rate

In rare cases, the CoinDesk Data API might return 0.00 as the exchange rate; In this case, PCG will show a warning and log the problematic record in the main window; This will obviously impact calculation accuracy
